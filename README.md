# ViNet_AirSim_drone
ViNet: Визуально-инерциальная одометрия (VIO) для навигации квадрокоптера в AirSim

## Описание проекта
Это нейросетевой пайплайн для оценки траектории движения квадрокоптера на основе синхронизированных данных монокулярной камеры и 6-осевого IMU. Модель обучается в высокореалистичном симуляторе **Microsoft AirSim** и предсказывает относительные смещения (delta pose) между кадрами, минимизируя накопление ошибки за счёт композитной функции потерь.

Проект демонстрирует навыки работы с мультисенсорными данными, проектирования гибридных архитектур (CNN + RNN), кастомных loss-функций и 3D-визуализации траекторий.

## Стек технологий
1. **Язык** Python 3.8+
2. **Deep Learning** PyTorch, `torchvision`
3. **Обработка данных** OpenCV, Pandas, NumPy, `pickle`
4. **Симуляция** Microsoft AirSim (сбор данных: камера + IMU + GT позы)
5. **Визуализация** Matplotlib (3D-траектории)
6. **Окружение** CUDA/CPU auto-detection

## Данные и предобработка
### Источник данных
Для сбора данных использовался полётный симулятор AirSim, собраны сырые данные полета по различным случайным траекториям с аугментацией изображения и добавлением шума в IMU.

- Папка `dataset/run_1/` содержит:
  - `images/*.png` – кадры с бортовой камеры
  - `imu.csv` – сырые данные акселерометра и гироскопа (`ax, ay, az, gx, gy, gz`)
  - `poses.csv` – ground truth позиции (`x, y, z`) в системе координат AirSim (NED)
Данные собирались в скриптом через AirSim API (record.py)
 
### Пайплайн обработки
1. **Временная синхронизация**: Привязка IMU-семплов к интервалам между кадрами через поиск ближайших соседей по временным меткам.
2. **Согласование систем координат**: Конвертация `NED → ENU` (инверсия оси Z для соответствия правосторонней системе).
3. **Нормализация**: Изображения ресайзятся до `224×224`, приводятся к `float32` и масштабируются `[0, 1]`.
4. **Формирование последовательностей**:
   - Окно кадров: `T = 5`
   - Фиксированная длина IMU-пакета: `20` шагов (с паддингом нулями при недостатке данных)
5. **Целевая переменная**: Относительное смещение `Δp = p_{t+1} - p_t` в ENU.

## Архитектура модели:
1. **Визуальный энкодер** | `ResNet18` (pretrained) → извлекает 512D-признаки для каждого кадра
2. **Инерциальный энкодер** | `LSTM(6 → 64)` → кодирует динамику IMU между кадрами
3. **Раннее слияние (Fusion)** | `Concat(CNN, IMU) → FC(512+64 → 256)` → попарное слияние модальностей
4. **Временное моделирование** | `LSTM(256 → 256)` → учитывает контекст всей последовательности `T=5`
5. **Выходной слой**  | `Linear(256 → 3)` → предсказание `(Δx, Δy, Δz)`

### Входы / Выходы
Входы:
  img_seq  : (Batch, T, 3, 224, 224)  # RGB кадры
  imu_seq  : (Batch, T, 20, 6)         # IMU-пакеты между кадрами

Выход:
  pred     : (Batch, T, 3)             # Предсказанные относительные смещения

## Функция потерь и обучение
Для борьбы с дрейфом (drift) одометрии используется двухкомпонентная функция:
L = L_{local} + 0.5 \cdot L_{trajectory}

где:
L_local – MSE на уровне шага: стабилизирует обучение и ускоряет сходимость.
L_trajectory – MSE на накопленной траектории (cumsum): штрафует модель за расхождение с GT на длинных горизонтах.

## Гиперпараметры
Оптимизатор - Adam
Learning Rate - 1e-4
Batch Size - 8
Epochs - 30

## Оценка и метрики
Предсказанные дельты интегрируются кумулятивно: pos_{t} = pos_{t-1} + Δp_{t}
Построенная траектория сравнивается с Ground Truth в 3D-пространстве.

## Ключевые результаты
Финальный Training Loss - 0.05371
Финальный Val Loss - 0.17643

## Итог:
Предсказанная траектория воспроизводит форму пути квадрокоптера и реагирует на повороты и ускорения. Это подтверждает, что модель обучилась извлекать признаки движения из визуального и инерциального потоков.
На графиках видно, что траектория смещается из за накопления ошибки на каждом шаге смещения. Это происходит по таким причинам:
- предсказанные Δp суммируются кумулятивно (cumsum). Любая малая ошибка на шаге t переносится на все последующие шаги, что даёт экспоненциальный дрейф
- отсутствие коррекции смещений IMU, данная модель не предсказывает и не компенсирует bias
- отсутствие информации о глубине, это неоднозначность масштаба.
- короткое временное окно, долгосрочные зависимости (возврат в ранее посещённую зону) не учитываются, нет механизма "памяти" для коррекции.

В последующих проектах данный пайплайн будет использоваться как дополнительная оценка движения квадрокоптера, например в связке с SLAM системами при потере ключевых точек.
